# Last Mile Studio UI 设计系统

状态：未来阶段的设计契约；不是实现方案

产品姿态：精准创意工作站

变化度：低

动效：低

密度：高、可读、可键盘访问

## 设计论点

Last Mile Studio 应该让人感觉像一件用于精确工作的工具：画布优先，源码始终权威，上下文靠近正在编辑的对象，全局操作与局部操作在视觉上分离。系统借鉴 Figma、Framer、VS Code 和 Linear 的纪律性，但不模仿其中任何一个产品。

UI 不是仪表盘。不需要装饰性卡片、营销 Hero、云端/账户外壳或推测性的协作功能。表面差异、1 像素边框、精确对齐和一个功能性强调色，应该完成几乎所有视觉工作。

规范状态仍为 `SourceDocument.document`。画布仍是标准 HTML/SVG 的经清理派生物。UI 组织可以改变；除非经过单独的产品契约阶段，否则序列化、稳定的 `data-editor-id`、选择语义、历史记录、导出、片段存储和源码同步不得改变。

## 信息架构

### 活动栏决策

活动栏是合理的，但只能由现有能力支撑。使用 40–42 px 的活动栏，并提供三个目的地：

1. **Layers**——结构树和图层操作。
2. **Pages**——文档具备多页面能力时显示或启用；提供缩略图和页面管理。
3. **Fragments**——临时剪贴板和本地库入口、搜索/过滤以及插入工作流。

除非存在真实的对应产品界面，否则不要添加 History、Search、Assets、云同步、协作、账户或设置目的地。撤销/重做不是 History 面板。“Fragments”可以使用可访问标签“片段与资源”，但不得暗示产品支持并不存在的通用资源系统。

活动栏选择一个左侧上下文面板的内容。它是导航，不是第二个模型：面板通过现有命令读取和修改规范文档。

### 区域职责

| 区域 | 负责 | 不得负责 |
|---|---|---|
| 顶部栏 | 身份、导入/打开、撤销/重做、预览、导出 | 选择属性、画布几何、页面/Build 编排 |
| 活动栏 | 在 Layers、Pages、Fragments 之间切换 | 不受支持的目的地，或自行修改文档 |
| 左侧上下文面板 | 一次负责一个导航/管理上下文 | 重复的检查器控件 |
| 画布上下文栏 | 当前模式、紧凑页面/Build 播放、画布尺寸/缩放；适用时显示选择条 | 完整页面管理或 Build 编排 |
| 画布 | 经清理的视觉派生物和直接操作 | 私有 UI 真值模型 |
| 右侧检查器 | Design、Build、Advanced 上下文编辑 | 全局导入/导出或重复的页面导航 |
| 底部状态栏 | 文档、选择、同步状态 | 主要操作 |
| 源码抽屉 | 定位/搜索/格式化/应用规范源码 | 没有明确边界的自动应用 |
| 对话框/弹出层 | 短暂、打断式、任务有界的工作流 | 永久导航或重复的核心控件 |

### Pages、Layers、Build、Inspector、Fragments 与 Source 的共存

- **Layers、Pages 和 Fragments** 是由活动栏选择的互斥左面板上下文。
- **Pages** 负责缩略图、复制、重排和删除。保留 `#page-filmstrip` 和 `#page-thumbnails`，但该界面可以在视觉上移入左侧上下文。多页面导航相关的情况下，中心仅保留上一页/当前页/下一页。
- **Build** 不是活动栏目的地。Build 编排属于右侧检查器的 Build 分组，因为它与页面/选择上下文相关。中心仅保留上一状态、当前状态、下一状态和播放视图模式。
- **Inspector** 是右侧容器，而不是重复的目的地。它包含 Design、Build 和 Advanced 分组。
- **Fragments** 负责左侧上下文中的发现/插入。保存选择和依赖/报告流程可以保留为对话框，因为它们是有边界的事务。
- **Source** 仍是底部抽屉，默认折叠。它不是活动栏目的地，也不是永久竞争性的编辑面板。

### 命令范围

| 范围 | 控件 |
|---|---|
| 全局 | 导入/打开、撤销、重做、预览、导出 |
| 模式特定 | 页面上一页/当前页/下一页、Build 播放、画布预设/尺寸、缩放/适应 |
| 选择特定 | 对齐/分布、几何、排版、外观、锁定/可见性/z-order、保存片段 |
| 文档类型特定 | HTML/SVG 导出标签、页面/Build/演示文稿可用性、仅 SVG 属性、仅 HTML 演示控件 |

选择特定的对齐操作只在存在选择对象时显示为紧凑上下文条。不适用时，文档类型特定控件应消失，而不是在没有解释的情况下保持禁用。

## 视觉 token

Token 名称采用语义命名。以下值是目标起点；只有在对比度和截图证据支持时才可调整。

### 表面与颜色

| Token | 值 | 用途 |
|---|---|---|
| `--surface-canvas-shell` | `#0b0d12` | 应用背景和最深层外壳 |
| `--surface-panel` | `#11141b` | 永久面板和顶部栏 |
| `--surface-raised` | `#171b24` | 控件、分组行、菜单 |
| `--surface-active` | `#202633` | 悬停/激活/选中的中性表面 |
| `--border-subtle` | `#292f3c` | 永久分隔线 |
| `--border-strong` | `#3a4353` | 邻近焦点的边界、激活分组边缘 |
| `--text-primary` | `#e8edf7` | 主要文字 |
| `--text-secondary` | `#b8c2d2` | 标签和次要值 |
| `--text-muted` | `#8c96a8` | 辅助文字 |
| `--text-subdued` | `#8792a5` | 允许使用的最低永久文字颜色 |
| `--accent` | `#5b8cff` | 选择、焦点、激活状态、主要操作 |
| `--accent-strong` | `#78a2ff` | 深色表面上的强调文字/悬停 |
| `--success` | `#5fd7a0` | 已确认同步/成功状态 |
| `--warning` | `#e7cb83` | 非破坏性警告 |
| `--danger` | `#ff6e78` | 破坏性/错误状态 |

强调色只承担功能，不作装饰。普通文字必须达到 4.5:1，对大号文字或有意义的非文字 UI 边界必须达到 3:1。`--text-subdued` 是下限，不是默认值。

### 排版

使用现有的本地/系统无衬线字体栈和等宽字体栈；不新增字体依赖。

| 角色 | 尺寸/行高 | 字重 |
|---|---|---:|
| 应用标题 | 16 / 20 px | 600 |
| 面板/分区标题 | 14 / 18 px | 600 |
| 标准 UI/控件 | 12–13 / 16–18 px | 450–550 |
| 辅助/元数据 | 至少 11 / 14 px | 400–500 |
| 代码 | 至少 12 / 18 px | 400 |

大写眉题可选，但不得低于 11 px，并且只在有助于说明层级时使用。不要用字间距来掩盖过小、难以阅读的文字。

### 间距

基础序列：`4, 8, 12, 16, 20, 24` px。

- 4 px：图标/文字内部间隙、紧密相关的元数据。
- 8 px：控件间距和紧凑行内边距。
- 12 px：分组内边距和标准垂直节奏。
- 16 px：面板分区之间的间隔。
- 20–24 px：对话框和空状态构图。

### 圆角

- 4 px：输入框、按钮、紧凑行；
- 6 px：菜单、分组、弹出层；
- 8 px：对话框和特殊浮动表面。

永久表面不得使用更大的圆角。

### 层级

永久面板使用表面值加 1 px 边框，不使用阴影。菜单/弹出层使用一层克制的阴影；对话框可以使用最强的层级。画布选择阴影属于已编辑对象/选择反馈，不属于外壳装饰。

建议 token：

- `--shadow-popover: 0 8px 24px rgb(0 0 0 / 0.32)`
- `--shadow-dialog: 0 18px 48px rgb(0 0 0 / 0.42)`

### 动效

| Token | 时长 | 用途 |
|---|---:|---|
| `--motion-fast` | 120 ms | 悬停/焦点颜色和透明度 |
| `--motion-standard` | 160 ms | 面板展开和小型状态过渡 |
| `--motion-emphasis` | 180 ms | 对话框/弹出层进入 |

尽可能只对透明度、变换和颜色做动效。调整大小和已编辑几何属性应直接更新。在 `prefers-reduced-motion: reduce` 下，非必要动画时长变为零，画布/图层自动滚动避免平滑行为。

## 布局 token 与桌面行为

| 视口 | 顶部栏 | 活动栏 | 左面板 | 右检查器 | 源码折叠 | 预期中心宽度 |
|---|---:|---:|---:|---:|---:|---:|
| 1280×800 | 48 px | 42 px | 216 px | 272 px | 40 px | 边框前约 748 px |
| 1440×900 | 48 px | 42 px | 240 px | 288 px | 40 px | 边框前约 868 px |
| 1920×1080 | 48 px | 42 px | 280 px | 320 px | 40 px | 边框前约 1276 px |

左右面板保持可调整大小和可折叠。现有画布最小尺寸保证必须保留：应用外壳可以重排或折叠，但不能修改已编辑文档的宽度、高度、坐标、字号或变换。

### 1280×800 紧凑桌面

- 全局操作保持可见。
- 左侧上下文默认 216 px，检查器默认 272 px。
- 中心使用一行紧凑模式。画布尺寸可以折叠到带标签的弹出层；缩放/适应保持可见。
- 选择对齐显示在第二个上下文行或受控溢出菜单中；不得产生文档级水平滚动。
- Pages 在左面板中纵向显示。不使用水平胶片条占用画布高度。
- 检查器分区使用折叠面板；Advanced 默认折叠。
- 源码展开高度为 `clamp(240px, 32vh, 360px)`，并保留至少 360×220 px 的可用画布视口。

### 1440×900 标准桌面

- 左侧上下文为 240 px，检查器为 288 px。
- 页面/Build 播放和缩放适应放在模式行；选择操作使用上下文条。
- Design 初始打开；对具备能力的 HTML/deck 文档显示 Build；Advanced 折叠。
- 在任何必需场景中，都不能出现控件相交、被裁切或造成外壳级水平溢出。

### 1920×1080 宽屏桌面

- 左侧上下文为 280 px，检查器为 320 px。
- 额外空间属于画布，而不是更大的永久面板或膨胀的控件。
- 顶部栏保留分组的全局操作；不为了填满空间而将控件铺满整行。
- 检查器可以同时显示更多分组，但控件尺寸和字号不放大。

## 控件规格

### 按钮

- 标准高度：30 px；紧凑密集高度：28 px。
- 水平内边距：8–10 px。
- 纯图标按钮：至少 28×28 px、16×16 px 图标、可访问名称、原生 tooltip 或一致的本地 tooltip。
- 主要样式仅用于导入/打开入口、导出、应用源码以及模态框中的肯定操作。一个界面不得包含多个视觉上同等的主要操作。
- 破坏性操作只在决策点使用危险色，并且不可逆时需要适当确认。

### 输入框、选择框和文本域

- 单行控件高度：30 px。
- 文字：至少 12 px；标签保持可见并以程序方式关联。
- 单位显示为相邻文字或输入装饰，而不是只放在 placeholder 中。
- 颜色输入及其文字值分别获得明确且唯一的标签。
- 无效状态同时使用边框和文字/图标；不能只靠颜色表达。

### 分段控件和选项卡

用于播放/分组/全部等互斥模式，不用于无关操作。按情况使用 `aria-pressed` 或选项卡语义。改变检查器分组可见性时，不得无意中重置待提交的值、选择状态、滚动位置或规范源码状态。

### 树和列表行

- 行高：28–32 px。
- 图层树实现 roving tabindex，并记录 Arrow/Home/End/Left/Right/Enter/Space/F2 行为。
- 保留拖动，但重排/重新挂载需要完整的键盘命令路径和结果播报。
- 选中、聚焦、锁定、隐藏和放置目标状态不能只靠颜色区分。

### 调整大小控件

保留语义化 separator 角色、方向、`tabindex="0"`、值属性、指针拖动和 Arrow 键调整大小。可见操作区可以很窄，但交互命中区至少横跨 8 px，并显示 `:focus-visible`。

### 菜单、弹出层和对话框

- 菜单保持附着于触发器，并留在视口内。
- 对话框限制过度滚动，提供可见标题，在安全时支持 Escape 关闭，并将焦点返回打开它的控件。
- 长内容在对话框内部滚动，而不是在其背后滚动。
- 空状态说明为空的内容，并提供一个相关的下一步操作。

## 交互状态系统

| 状态 | 契约 |
|---|---|
| 默认 | 标签清晰、对比度足够、不做不必要的边框强调 |
| 悬停 | 细微的抬高表面或文字变化；不移动布局 |
| 焦点 | 2 px 强调色环，并带 2 px 偏移或等效内/外描边；在 `:focus-visible` 下始终可见 |
| 激活/选中 | 强调色边缘/填充，加上图标/勾选/字重等第二提示 |
| 禁用 | 降低强调程度但保持可读；保留原生语义，必要时提供解释 |
| 加载 | 保持尺寸；显示本地化进度；不阻塞无关编辑工作 |
| 成功/已同步 | 简短、平静的状态；只播报有意义的变化 |
| 错误 | 在受影响控件旁解释原因和恢复方式；保留用户输入 |
| 空 | 说明缺少什么以及唯一最相关的操作；不需要装饰性插图 |

## 图标策略

- 使用小型本地内联 SVG 辅助组件或共享 SVG symbol 集；不新增生产依赖。
- 标准图标画布为 16×16，统一使用 1.5 px 描边，并在适当情况下使用圆角线帽/连接。
- 图标继承 `currentColor`。
- Unicode 字形不得成为操作唯一含义。
- 每个纯图标控件都需要可访问名称和 tooltip。
- 一个命令族内不要混用填充、描边、emoji 和文字符号风格。

## 应避免的反模式

- 不迁移框架，不引入 Tailwind 或新的生产依赖。
- 不创建私有 UI 文档模型，不使用视口推导已编辑几何属性。
- 不使用玻璃拟态、霓虹/赛博朋克配色、装饰性的蓝紫渐变或画布外网格装饰。
- 不使用过大的圆角、永久面板阴影、每个设置都包一张装饰性卡片，或以牺牲密度为代价的空白膨胀。
- 不使用只在悬停时出现的必要操作、神秘图标、无标签输入框，或为了“塞下”拥挤布局而使用过小文字。
- 不为不受支持的能力设置活动栏目的地。
- 不因为空间存在就重复永久控件。
- 不为了适应标记而弱化测试、无谓更换选择器或重置存储。
- 不自动应用源码编辑，不执行导入的脚本/事件处理器。
