# Last Mile Studio UI 验收契约

状态：面向未来实现阶段的可度量验收标准

## 证据模型

验收需要三类相互独立的证据：

1. **行为 oracle**——现有单元测试和浏览器 smoke 证明编辑、源码同步、历史记录、页面、Build、片段、预览、导出和布局行为保持正确。通过截图不能替代这个 oracle。
2. **结构指标**——自动化 DOM/浏览器测量证明名称、目标尺寸、溢出、裁切、重复 ID、运行时错误和稳定钩子。指标不能决定构图是否清晰。
3. **视觉判断**——人工检查截图中的层级、密度、节奏、可读性、状态清晰度和工作站气质。视觉上好看的截图不能豁免失败的 oracle 或指标。

只有当所列的每种证据都通过时，一个标准才算完成。不得为了让重设计后的标记通过而弱化现有测试。

## 行为验收

| ID | 标准 | 证据与映射 |
|---|---|---|
| A-BEH-01 | `SourceDocument.document` 仍是唯一规范编辑状态；画布和检查器保持为同步的派生物。 | 行为 oracle：当前模型/编辑器单元测试和 `npm test`；浏览器序列 S-01。 |
| A-BEH-02 | 标准 HTML/SVG 序列化、稳定的 `data-editor-id`、选择身份、历史记录和导出往返保持不变。 | 行为 oracle：现有序列化/历史/导出测试和浏览器 smoke；S-01、S-07。 |
| A-BEH-03 | 不执行导入脚本、内联事件处理器和不可信本地命令。 | 行为 oracle：现有清理/导入测试；安全断言保持完整。 |
| A-BEH-04 | 切换 Layers、Pages 或 Fragments 上下文不会改变文档、选择、历史记录、视口所编辑几何属性或未保存字段值。 | 新的聚焦 UI 测试以及 S-02/S-03/S-05。 |
| A-BEH-05 | 页面选择、缩略图选择、上一页/下一页、复制、重排、删除以及持久化活动页保持当前语义。 | 现有页面/浏览器测试以及 S-03。 |
| A-BEH-06 | Build 分配、分组、重排、播放上一项/下一项、视图模式和预览行为保持正确。 | 现有 Build/演示测试以及 S-04/S-06。 |
| A-BEH-07 | 片段保存、剪贴板、本地库、过滤、插入、依赖报告和存储行为保持正确。 | 现有片段测试以及 S-05。 |
| A-BEH-08 | 源码抽屉初始折叠，折叠高度 ≤44 px，切换文字保持 `展开源码` / `收起源码`，Apply 仍是明确操作。 | 浏览器 smoke 以及 S-07；对初始和展开状态进行结构指标检查。 |
| A-BEH-09 | 调整大小控件支持指针和键盘操作；折叠的宽度/高度以及偏好在重新加载后保持。 | 现有布局测试以及 S-08；schema 变更时增加存储迁移测试。 |
| A-BEH-10 | 应用外壳视口变化不会修改已编辑画布尺寸、坐标、排版、变换或序列化源码。 | 三种视口的 Canvas gate；S-09 中前后序列化文档相等。 |

## 结构验收

| ID | 可度量标准 | 所需状态 |
|---|---|---|
| A-STR-01 | 可见 ID 无重复，现有功能所需的受保护 ID/钩子无缺失。 | 所有视口下的每个审计场景。 |
| A-STR-02 | 每个可见 input、select、textarea、button、summary 操作和纯图标控件都有唯一的可访问名称。 | 默认、选中、deck、SVG、源码、片段、预览。 |
| A-STR-03 | 标准控件高度为 28–32 px；每个纯图标按钮命中区至少为 28×28 px。 | 所有永久和临时界面。 |
| A-STR-04 | 交互文字至少 12 px；辅助/元数据至少 11 px；代码至少 12 px。 | 所有场景的计算样式审计。 |
| A-STR-05 | 普通文字对比度至少 4.5:1；有意义的非文字 UI 边界/焦点至少 3:1。 | token 测试以及代表性的计算样式/对比度审计。 |
| A-STR-06 | 应用外壳没有水平滚动，必要控件没有被裁切到视口之外。 | 1280×800、1440×900、1920×1080 的所有场景。 |
| A-STR-07 | 紧凑桌面下，即使源码展开以及处于 deck/选中上下文，画布视口仍至少为 360×220 px。 | 紧凑源码、deck、选中状态。 |
| A-STR-08 | 永久面板尺寸与目标默认值相差不超过 ±2 px：活动栏 42，左侧 216/240/280，右侧 272/288/320，源码折叠 40。 | 每个目标视口的默认状态。 |
| A-STR-09 | 每个交互类别都具有可感知的 focus-visible，包括活动栏项目、树行、调整大小控件、画布、工具栏、检查器、源码和对话框。 | 键盘遍历截图以及 CSS/状态审计。 |
| A-STR-10 | 减少动效模式会移除非必要动画；不得引入影响布局的动画。 | `reducedMotion: reduce` 的浏览器上下文；CSS 审计。 |
| A-STR-11 | 不新增浏览器页面、控制台或请求错误。 | 每个浏览器场景和 release gate。 |
| A-STR-12 | 状态更新使用已记录的 live-region 策略，不在指针移动/调整大小期间持续播报。 | 可访问性树检查以及 S-01/S-08。 |
| A-STR-13 | 对话框位于视口内、包含过度滚动、适当关闭，并将焦点返回触发器。 | 粘贴、片段、预览选择、演示、报告对话框。 |
| A-STR-14 | 布局偏好变更保持或迁移 `last-mile-studio:layout:v1`；不得静默重置。 | 重新加载/迁移测试。 |

## 视觉验收

| ID | 判断标准 | 失败示例 |
|---|---|---|
| A-VIS-01 | 在每个目标视口中，画布都是主要视觉区域。 | 永久面板扩展到占满宽空间；Pages/源码不必要地消耗画布高度。 |
| A-VIS-02 | 全局、模式和选择命令在视觉上有区别，但没有重复。 | 页面管理出现在两个地方；Build 编排同时出现在中心和检查器；选择工具始终可见。 |
| A-VIS-03 | Layers、Pages 和 Fragments 在同一个导航系统中作为目的地呈现；不出现不受支持的能力。 | 虚假的 History/云端/账户目的地；没有标签/tooltip 的含义不明图标。 |
| A-VIS-04 | 检查器表达 Design、Build 和 Advanced 层级，并将常用属性放在高级字段之前。 | 一个没有分化的表单流；Build 在非 deck 选择中永久占主导。 |
| A-VIS-05 | 密集控件保持可读且对齐；不使用 7–10 px 文字来强行适配。 | 标签被截断；动作行压缩到 20 px；元数据对比度过低。 |
| A-VIS-06 | 表面是克制的中性深色，并使用一个功能性蓝色强调色。 | 装饰性渐变、霓虹光晕、玻璃面板、大型永久阴影、装饰性卡片。 |
| A-VIS-07 | 焦点、悬停、选中、禁用、加载、成功、警告、错误和空状态可区分且一致。 | 只用颜色表示选择；禁用文字变得不可读；空对话框没有下一步。 |
| A-VIS-08 | 宽布局把额外空间留给画布，而不是扩大面板、间距或字号。 | 控件在未使用的顶部栏宽度中散开；检查器无限增长。 |

## 必需截图矩阵

每个相关阶段后，使用确定性的样例数据、清空的 local/session storage、深色配色、`zh-CN` 和减少动效来捕获 PNG。前后文件必须使用相同的视口和场景。

| 场景 | 1280×800 | 1440×900 | 1920×1080 | 必需评审 |
|---|---:|---:|---:|---|
| 默认 HTML | 是 | 是 | 是 | 全局层级、画布主导地位、无空白外壳膨胀 |
| 选中 HTML 元素 | 是 | 是 | 是 | 选择条、Design 检查器、标签、焦点 |
| 多页面 deck | 是 | 是 | 是 | Pages 上下文、Build 可用性、工具栏无溢出 |
| 左侧上下文折叠的 deck | 是 | 是 | 是 | 画布恢复和紧凑模式控件 |
| SVG 选择 | 是 | 是 | 是 | 文档类型控件和颜色/输入标签 |
| 源码展开 | 是 | 是 | 是 | 对 Apply 的明确表达、代码尺寸、最小画布区域 |
| 片段库 | 是 | 是 | 是 | 密度、空/过滤状态、对话框焦点 |
| 预览选择 | 是 | 是 | 是 | 对话框层级和焦点 |
| 演示打开 | 是 | 是 | 是 | 实际的 `#presentation-dialog[open]`，而不是只有预览选择 |
| 键盘焦点序列 | 是 | 是 | 是 | 活动栏/树/画布/检查器/源码中的可见焦点 |

截图文件名应保持确定性，例如 `compact__deck.png`，并且报告必须链接每个必需路径。

## 必需交互序列

### S-01——选择、编辑、源码同步、撤销/重做

1. 加载 HTML 示例。
2. 先从画布选择一个元素，再从 Layers 选择它。
3. 修改一个几何值和一个外观值。
4. 确认画布和源码同步，以及一个历史边界。
5. 使用键盘撤销和重做。
6. 确认 `#document-status`、`#selection-status` 和 `#sync-status` 准确，并以适当方式播报。

通过 A-BEH-01、A-BEH-02、A-STR-02、A-STR-12。

### S-02——纯键盘图层树

1. 聚焦 Layers 活动栏目的地和树。
2. 使用 Arrow/Home/End 导航，使用 Left/Right 折叠/展开或移动层级焦点，使用 Enter/Space 选择，使用 F2 重命名。
3. 使用文档化的键盘替代方案重排或重新挂载。
4. 确认焦点保持可见、画布选择跟随，并播报结果。

通过 A-BEH-04、A-STR-03、A-STR-09。

### S-03——Pages 上下文

1. 加载 deck 示例并从活动栏打开 Pages。
2. 通过缩略图选择，使用上一页/下一页、复制、重排和带确认语义的删除。
3. 折叠/重新打开左面板并重新加载。
4. 确认活动页、页面顺序、选择和已编辑几何属性保持正确。

通过 A-BEH-04、A-BEH-05、A-BEH-09、A-BEH-10。

### S-04——Build 编排与播放

1. 选择一个元素，并在右侧检查器中将其分配给一个 Build 组。
2. 重排分组/元素并检查警告。
3. 切换播放/分组/全部视图模式；使用指针和键盘进行上一项/下一项。
4. 确认中心只显示播放，不重复编排控件。

通过 A-BEH-06、A-VIS-02、A-VIS-04。

### S-05——片段

1. 将选择保存为片段；取消一次并确认焦点返回。
2. 从 Fragments 打开临时剪贴板和本地库。
3. 过滤/搜索，在支持时收藏，检查依赖报告并插入。
4. 确认源码清理、身份、存储、空/错误/加载状态和键盘操作。

通过 A-BEH-03、A-BEH-04、A-BEH-07、A-STR-13。

### S-06——演示预览

1. 激活 `#preview-presentation`。
2. 确认 `#preview-choice-dialog` 打开，焦点进入其中。
3. 选择起始页/当前页；确认 `#presentation-dialog[open]`，并捕获该状态。
4. 使用控件关闭，并在安全时使用 Escape；确认焦点返回 Preview。

通过 A-BEH-06、A-STR-13，并防止基线 harness 不匹配。

### S-07——源码优先工作流

1. 确认抽屉初始折叠、高度 ≤44 px，并显示 `展开源码`。
2. 展开；定位选择、搜索、格式化，进行一次有效编辑并明确应用。
3. 确认脏数据/错误/同步状态、画布更新和历史行为。
4. 折叠并确认画布高度恢复，且只有展开时标签才变为 `收起源码`。

通过 A-BEH-01、A-BEH-08、A-STR-07。

### S-08——布局持久化和调整大小控件

1. 通过指针调整左侧、右侧和适用的水平区域。
2. 使用键盘以普通步长和 Shift 步长调整每个分隔线。
3. 折叠/重新打开面板并重新加载。
4. 确认持久化状态、`aria-valuenow/min/max`、焦点可见和画布恢复。

通过 A-BEH-09、A-STR-09、A-STR-14。

### S-09——视口不变性

1. 在 1440×900 下记录规范源码以及已编辑的画布/页面尺寸。
2. 在 1280×800 和 1920×1080 下重复，包括面板折叠和源码展开。
3. 比较序列化源码、坐标、尺寸、字号和变换。
4. 只接受外壳重排；拒绝任何已编辑文档变更。

通过 A-BEH-10、A-STR-06、A-STR-07、A-VIS-01、A-VIS-08。

## 标准到门禁的映射

| 门禁 | 主要覆盖 |
|---|---|
| `bash scripts/ui-gate.sh fast` | 类型检查、单元行为 oracle、快速结构/视觉 smoke |
| `bash scripts/ui-gate.sh checkpoint` | 完整阶段浏览器检查点和必需截图 |
| `bash scripts/ui-gate.sh canvas` | 相关阶段后的画布/页面/布局/视口不变性 |
| `bash scripts/ui-gate.sh release` | 完整 oracle、结构指标、截图和运行时错误审计 |
| `git diff --check` | 仅检查补丁卫生；不能证明产品验收 |

## 发布决策规则

只有在以下条件全部满足时，重设计才算完成：

- 所有适用的 A-BEH 标准通过，且没有弱化断言；
- 所有 A-STR 标准在三个目标视口下通过；
- 人工评审接受所有 A-VIS 标准和截图；
- 没有未解决的 P0/P1 发现；
- P2 延期项具有明确负责人和理由；
- 受保护选择器、钩子、存储、安全边界和源码优先架构均已验证；
- 最终差异没有无关清理。

如果同一个修复失败两次，回滚最小的失败补丁，缩小阶段范围并报告阻塞。不得通过放宽标准或审计来解决失败的验收标准。
